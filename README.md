## 游戏介绍

这是一个基于 HTML5 Canvas 的经典塔防游戏。玩家通过在地图上建造各种防御塔来阻止怪物到达终点。游戏包含三种不同类型的防御塔和三种怪物，支持塔升级系统、波次预览、特殊效果等功能。

**核心特性：

- 3 种防御塔：速射塔、炮塔、冰霜塔，各有独特属性和升级路线
- 3 种怪物：普通怪、快速怪、坦克怪，属性各不相同
- 塔升级系统：每座塔最多可升级 3 级，提升伤害、射程等属性
- 波次预览：开始下一波前可看到怪物类型和数量
- 怪物 grace period：怪物到达终点后有 600ms 宽限期，塔仍可攻击
- 减速效果：冰霜塔可减速敌人，并有视觉特效
- 范围伤害：炮塔有溅射伤害，可同时伤害多个敌人

## 游戏玩法

### 操作方式

| 操作 | 说明 |
|------|------|
| 左键点击右侧塔按钮 | 选择要建造的塔类型 |
| 左键点击草地 | 放置防御塔（需选中塔类型时） |
| 左键点击已放置的塔 | 选中塔，查看属性和升级 |
| 右键 | 取消选择 |
| 点击「开始波次」按钮 | 开始下一波怪物 |

### 游戏规则

- 初始金币：150
- 初始生命：20
- 怪物沿固定路线移动，到达终点后扣除 1 点生命
- 怪物到达终点后有 600ms 宽限期，塔仍可攻击
- 击杀怪物获得金币和得分
- 金币可用于建造和升级防御塔
- 生命为 0 时游戏结束

### 得分计算

- 击杀怪物得分 = 怪物奖励金币 × 2

## 波次系统

### 波次生成规则

- 每波怪物基础数量 = 5 + 波次数 × 1.5（向下取整）
- 第 1-2 波：只有普通怪
- 第 3-4 波：70% 普通怪，30% 快速怪
- 第 5 波及以后：50% 普通怪，30% 快速怪，20% 坦克怪
- 怪物生成间隔：800ms

### 波次预告

每波开始前，右侧面板会显示下一波的怪物类型和数量预览。

## 测试说明

### 运行测试

在浏览器中打开 `test.html`，点击「运行测试」按钮即可运行所有测试。

或者在浏览器控制台中执行：

```javascript
const summary = runAllTests();
console.log(summary);
```

### 测试覆盖

当前共 **32** 个测试用例，覆盖以下模块：

#### 1. 塔升级逻辑（8 个用例）

- 速射塔初始属性验证
- 速射塔 Lv1~Lv3 升级属性验证
- 速射塔满级不能继续升级
- 炮塔各级伤害和范围验证
- 冰霜塔各级减速效果验证
- 升级花费验证

#### 2. 怪物 grace period（5 个用例）

- 到达终点进入 grace period
- 到达终点后超过 600ms 扣命
- grace period 内塔能打到敌人
- 到达终点后敌人不移动
- 超过 grace period 后塔不能打到

#### 3. 金币和得分计算（12 个用例）

- 初始金币和得分验证
- 击杀普通怪/快速怪/坦克怪得金币和得分
- 建造速射塔/炮塔/冰霜塔扣费验证
- 升级扣费验证（多次升级）
- 钱不够不能建造
- 钱不够不能升级
- 到达终点不给金币
- grace period 内被击杀不扣命

#### 4. 波次预告数据（7 个用例）

- 波次 1 预告数据结构正确
- 波次 1 预告与实际生成一致
- 波次 5 预告包含快速怪
- 波次 10 预告包含坦克怪
- 波次预告包含类型、数量、奖励、血量
- 预告数量大于 0
- 预告类型在合法范围内

### 测试框架

项目使用自研轻量级测试框架，位于 `js/tests.js`，包含：

- `describe(name, fn)` - 测试套件
- `test(name, fn)` - 测试用例
- `assertEqual(actual, expected, message)` - 相等断言
- `assertTrue(actual, message)` - 真值断言
- `assertFalse(actual, message)` - 假值断言
- `assertCloseTo(actual, expected, delta, message)` - 近似值断言
- `assertLessThan(actual, expected, message)` - 小于断言
